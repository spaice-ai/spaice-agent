# SPAICE Agent Framework

Pre-turn middleware for Hermes-based agents — memory recall, search, and
on-demand 3-chair consensus. Install with one line; ship updates with one
command.

```bash
curl -sSL https://spaice.ai/install.sh | sh -s <agent_id>
```

## What it does

On every user turn, before the LLM sees the message, the framework:

1. **Recalls memory** — scans an obsidian-like vault for entities
   mentioned in the message, injects top matches as context.
2. **Fires search** when trigger phrases hit (`look up`, `latest`, etc.)
   — runs Exa + Brave in parallel, RRF-merges, optionally short-circuits
   the reply.
3. **Emits a consensus advisory** when architecture/decision trigger
   phrases hit — nudges the LLM to call the `use_consensus` tool if it
   actually wants a 3-chair panel. Cheap (no inline consensus).
4. **Registers the `use_consensus` tool** — LLM-callable; runs
   Opus-4.7 → GPT-5-codex → DeepSeek pipeline when invoked. Cost
   ~$0.15-0.30, latency 30-90s.

All business logic lives in the `spaice_agent` Python package. The
Hermes hook is a 13-line autogenerated shim. Bugs fix once, ship to
every installed agent via `spaice-agent upgrade`.

## Install (one line)

```bash
curl -sSL https://spaice.ai/install.sh | sh -s jarvis
```

Replace `jarvis` with your agent ID (lowercase alphanumeric + hyphens).
You can install multiple agents side-by-side — each gets its own
namespace under `~/.spaice-agents/<id>/`.

Pin a version:

```bash
curl -sSL https://spaice.ai/install.sh | sh -s jarvis v0.1.0
```

## CLI

```
spaice-agent install <id>         Write hook shim to ~/.Hermes/hooks/
  --with-config                   Also scaffold ~/.spaice-agents/<id>/config.yaml
  --force                         Overwrite without backup
spaice-agent uninstall <id>       Remove hook (keep config)
  --purge                         Also remove config + state
  --keep-backup                   Rename instead of delete
spaice-agent list                 Show all installed agents
spaice-agent upgrade              Reinstall package + refresh every shim
spaice-agent doctor <id>          8-point health check
spaice-agent version              Print installed version
```

## Filesystem layout

Each installed agent uses:

| Path | What |
|---|---|
| `~/.Hermes/hooks/spaice-<id>/handler.py` | 13-line autogenerated shim |
| `~/.Hermes/hooks/spaice-<id>/HOOK.yaml` | Hermes hook manifest |
| `~/.Hermes/hooks/spaice-<id>/.spaice-install.json` | Install version manifest |
| `~/.spaice-agents/<id>/config.yaml` | Agent config (memory root, triggers, consensus pipeline) |
| `~/.spaice-agents/<id>/state/` | Suppression counter, consensus ledger, budget ledger |

## After install

The installer scaffolds `config.yaml` with placeholders. Before the
agent does anything useful:

1. Edit `~/.spaice-agents/<id>/config.yaml` — set `memory_root` etc.
2. Add credentials to `~/.Hermes/credentials/`:
   - `openrouter.key` — for consensus
   - `exa.json` — for search
   - `brave.json` — for search
3. Restart Hermes so the hook loads
4. Verify: `spaice-agent doctor <id>` should report all 8 checks green

## Upgrade

```bash
spaice-agent upgrade
```

Does two things atomically:
1. `pip install --upgrade` the package from the pinned source
2. Rewrites every installed agent's hook shim with the new version

The doctor will warn if a shim is stale (`⚠`) — run upgrade to refresh.

## Architecture

```
User turn
    │
    ▼
~/.Hermes/hooks/spaice-<id>/handler.py   [13-line shim]
    │
    ▼  (imports)
spaice_agent.hook.make_hook(agent_id)
    │
    ├─► orchestrator.handle_message()
    │     ├─► triggers.scan() — keyword/phrase matchers
    │     ├─► memory_recall.recall() — vault scan
    │     ├─► search.fire() — Exa + Brave parallel
    │     ├─► advisory.is_suppressed() — counter check
    │     └─► advisory.build_advisory() — string builder
    │
    └─► register_tools(registry)
          └─► tools.use_consensus — 3-chair pipeline tool
                └─► consensus.run_consensus()
```

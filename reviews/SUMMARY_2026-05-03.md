## High-severity must-fix items

### config.py
- **Path traversal** — `agent_id="../aurora"` walks up the tree before regex check. Validate `agent_id` against `AGENT_ID_PATTERN` BEFORE constructing `config_path`. Breaks the "Jarvis/Aurora isolated" requirement.
- Medium: `search.enabled=true` with empty `providers` array should be rejected at validation time.
- Medium: `OSError`/`UnicodeDecodeError` not caught → raw exception instead of `ConfigError`.
- Medium: no `encoding="utf-8"` on file open.

### triggers.py
- **Short-trigger block** — `consensus_triggered` bails on `len < 20`, killing "review this", "audit logs", "plan rollout". Same for search at `len < 10`. Remove the length guards or drop them to much lower (e.g. 5).
- **Fenced-code false positive** — `re.sub(r"\`[^\`]*\`", ...)` closes on first inner backtick. A fenced block containing any literal backtick (common in template strings / markdown examples) leaves keywords exposed. Needs proper fence handling.
- Medium: pattern cache keyed only on `agent_id` → YAML edits don't flip behaviour without restart. Use `(agent_id, id(config))` or clear cache when a new config instance appears.
- Medium: malformed regex in YAML crashes the whole hook. Wrap compilation in try/except; fall back to "no matches" with a warning.

### budget.py
- **Racy cap enforcement** — `can_fire()` reads without lock, `increment()` takes lock separately. Two concurrent fires both see "29 < 30" and both increment to 30 and 31. Need atomic `check-and-increment` that holds the lock across the check AND the write.
- **Lost updates on lock timeout** — current fallback returns `count+1` without persisting. Next successful increment overwrites with stale data. Should raise so caller can abort, not pretend to succeed.
- Medium: `Ledger.record` drops writes on disk error → under-reports spend silently.
- Medium: `datetime.fromisoformat` pre-Py3.11 doesn't accept `Z` suffix. We target 3.11+ per pyproject, so only a concern if we support older runtimes.
- Medium: `BudgetExceeded` is declared but never raised by anything in this module.

### openrouter_client.py
- **Retry loop oblivious to deadline** — when caller wraps `chat()` in `asyncio.wait_for(3s)` and we get a 429 with `Retry-After: 2` plus linear backoff after, we'll happily sleep 3+ seconds before the outer deadline `CancelledError`s us. Need `deadline: float | None` parameter that the retry loop respects.
- **30s default timeout** — far above any hook budget. Caller must always override. Either drop default to something sane (5s) or require explicit.

## Decision for each

Fix ALL high-severity items before committing more modules. Medium items addressed where cheap. Tests extended to cover the failure modes that slipped through originally:

- `test_config_rejects_path_traversal_agent_id`

- `test_budget_lock_timeout_does_not_lose_updates`
- `test_openrouter_deadline_aware_retry`
